# 执行-提炼-沉淀-复用-自省 工作闭环

> 敬哥铁律：完成任务只是开始。"做完"不等于"完成"。

## 五步闭环（v2.0 — 融入静默观察模式）

```
观察 → 执行 → 提炼 → 沉淀 → 复用 → 自省
  ↑_____________________________|
```

### 第0步：观察（Observe — 参考 Skill Factory 模式）
**执行过程中静默跟踪模式**

在执行任务的同时，保持以下几点心智记录：

| 跟踪什么 | 示例 |
|---------|------|
| 重复动作 | 同一组命令/步骤用了2次+ |
| 多步工作流 | 3步以上完成一个连贯目标 |
| 工具组合 | 两个以上工具以固定顺序配合使用 |
| 临时修复 | 任何绕过/workaround |
| 用户纠正 | 用户说"不对""不是这样""换个方式" |

**不要在任务中间打断用户提问题。** 静默观察，等到合适的时机再提议。

### 第1步：执行（Execute）
**做对、做完、做透**

- 用户交代的事，先做出来
- 不打折扣、不找借口、不省步骤
- 做完了才有提炼的基础

### 第2步：提炼（Refine）
**做完后问自己三个问题：**

| 问题 | 含义 |
|------|------|
| 这个任务的核心方法是什么？ | 如果用一句话教别人，怎么说？ |
| 有没有踩坑？怎么避开的？ | 下次遇到同样问题怎么不走弯路？ |
| 能不能形成一套可重复的步骤？ | 下次再做，能不能直接套模板？ |

### 第3步：沉淀（Skill）
**把提炼的结果写成技能**

#### 何时提议/创建

| 条件 | 操作 |
|------|------|
| 用户明确说"记住这个""存为技能" | ✅ 立即创建 |
| 用户纠正了你的做法 | ✅ 必须沉淀 |
| 同一模式出现2次+ | ✅ 主动提议 |
| 任务用了5次以上工具调用 | ✅ 沉淀 |
| 踩了坑、花了时间才解决 | ✅ 沉淀 |
| 一次性问答 / 简单信息查询 | ❌ 不沉淀 |

#### 提议格式（参考 Skill Factory）

当检测到值得沉淀的模式时，用此格式向用户提议：

```
🏭 检测到可沉淀的工作流

我注意到你 [重复做了什么 / 纠正了什么]。

提议技能:   [技能名]
分类:       [分类]
说明:       [一句话说明]

捕获内容:
  1. [步骤1]
  2. [步骤2]
  3. [步骤3]

生成: [A] 仅SKILL.md  [B] 仅plugin.py  [C] 两者  [D] 跳过
```

#### 输出格式

**SKILL.md** — Hermes 标准技能文件

```markdown
---
name: 技能名称（一看就知道什么时候用）
description: 一句话说清楚解决什么问题
tags: [相关标签]
trigger: 什么场景下触发
---

# 技能名称

## 步骤
1. 第一步...
2. 第二步...
3. ...

## 踩坑记录
| 坑 | 怎么避开的 |
|---|-----------|

## 验证方法
做完后怎么确认做对了
```

**plugin.py**（可选）— 斜杠命令插件，参考 Skill Factory 模板生成。

#### 技能质量标准

生成的 SKILL.md 必须：
- ✅ 有具体步骤（不是模糊指引）
- ✅ 包含至少一个真实示例
- ✅ 定义清晰的触发条件
- ✅ 控制在 600 行以内
- ✅ 包含"为什么这样做"而不仅是"做什么"

#### 引用本会话的工具和方法

| 工具/技能 | 位置 | 用途 |
|----------|------|------|
| Token 压缩层 | ~/.hermes/scripts/token_compression.py | 工具返回大数据时先压缩再进上下文 |
| Auto-Fetch | ~/.hermes/scripts/auto_fetch.py | 每20分钟自动汇聚上下文快照 |
| Skill Factory 元技能 | ~/.hermes/skills/meta/skill-factory/SKILL.md | 自动检测工作流→提议创建技能 |
| SkillClaw | ~/.hermes/SkillClaw/ | 集体技能进化系统 |
```

### 第4步：复用（Reuse）
**下次遇到同类问题，先查技能库，不重新发明轮子**

- 接到任务时，**先加载相关技能**
- 确认技能是否还适用（有没有过时？有没有更好的方法？）
- 如果适用 → 直接复用
- 如果不完全适用 → 先修改技能再复用（用patch更新）

### 第5步：自省（Reflect）
**定期复盘，问自己：**

**任务结束后立即问：**
- 这事我做得好的地方是什么？
- 哪一步可以更快/更好？
- 我有没有让敬哥重复说同样的话？

**每周复盘：**
- 这周沉淀了几个技能？
- 有哪个技能用了之后发现不对？更新了吗？
- 有没有"做了就忘了"的事情？

## 反合理化表

| 借口 | 反驳 |
|------|------|
| "这个太简单了，不用记" | 简单的才更应该记，省得下次又从头想。 |
| "用户没说要沉淀" | 敬哥说了：每次任务完成都要走五步。不用等用户说。 |
| "先干下一件事，回头再沉淀" | "回头"是最大的谎言。做完当下立刻沉淀，过5分钟就忘了。 |
| "这个场景不常见" | 不常见才更要记，不然下次遇到完全想不起来怎么解决。 |
| "技能太多会混乱" | 技能不是越多越好，而是越精准越好。用分类归档，定期清理由 curator 处理。 |
| "敬哥只让我做这个，没要技能" | 敬哥让你执行-观察-提炼-沉淀-复用-自省。技能是必须的。 |
| "已经有类似技能了" | 先看那个技能是否完整覆盖了本场景。如果缺步骤，更新它；如果完全不同，新建。 |
| "这种模式只出现了一次，不够格" | 用户纠正过你，或者解决过程花了5步以上，就算一次也值得记录。 |

## 执行检查清单

- [ ] **执行** — 任务做完了，结果确认了
- [ ] **提炼** — 核心方法总结出来了
- [ ] **沉淀** — 已用 skill_manage(action='create') 或 skill_manage(action='patch') 写入技能
- [ ] **复用** — 确认下次可以调用这个技能
- [ ] **自省** — 完成了复盘，确认没有遗漏

> ❗ **五步缺一不可。不做完不算完成。**
