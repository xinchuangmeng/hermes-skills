# Agent Skills-as-Workflows 方法论

> 来源：Addy Osmani - Agent Skills (May 2026, 26K+ stars)

## 核心原则

一条好技能 = **工作流（workflow）**，不是参考文档（reference documentation）。

| ❌ 坏技能（参考文档） | ✅ 好技能（工作流） |
|----------------------|---------------------|
| "要写好spec，需要考虑..." | 步骤1：写用户故事 → 步骤2：定义AC → 步骤3：确认边界 |
| 长篇大论的解释性文字 | 有退出标准的检查点序列 |
| Agent读完后不知道该做什么 | Agent读完自动开始执行 |

## 五条设计原则

### 1. Process Over Prose（流程优于文字）
- 工作流是Agent可执行的；论文不是
- 200页手册没人读；带检查点的小流程会被执行
- **做法**：每条技能控制在5-10个可执行步骤

### 2. 反合理化表（Anti-Rationalization Tables）
Agent（和疲惫的人）总是会找借口跳过关键步骤。预写反驳。

**模板：**
```
| 借口 | 反驳 |
|------|------|
| "这个任务太简单，不需要spec" | AC（验收标准）仍然需要。5行也行，0行不行。 |
| "我先写代码，测试后面补" | "后面"是最承重的词。没有后面。先写失败的测试。 |
| "测试通过了，发吧" | 通过测试是证据，不是证明。检查runtime了吗？有真人review过diff吗？ |
| "看起来没问题" | "看起来"从来不是充分条件。 |
```

### 3. 验证不可协商
每条技能必须以**具体证据**终止：
- ✅ 测试通过了（具体哪个测试）
- ✅ 构建完成了（无警告）
- ✅ Runtime trace显示预期行为
- ✅ Reviewer签字确认

"看起来没问题" ⛔ → 永远不是充分条件。

### 4. 渐进式暴露（Progressive Disclosure）
- 不要一次性把所有技能加载到上下文
- 根据当前阶段激活相关技能
- 元技能（meta-skill）作为路由器

> "每个加载到上下文的token都会损害某些性能，所以只加载相关的，把剩下的留在磁盘上。"

### 5. 范围纪律（Scope Discipline）
> "只碰让你碰的东西。"

- 不要重构相邻系统
- 不要删除你不懂的代码
- 不要因为一个TODO就重写整个文件

**范围纪律是决定Agent的PR能否被merge的最重要因素。**

## 完整技能生命周期（SDLC）

| 阶段 | 命令 | 目的 | 适用技能数量 |
|------|------|------|-------------|
| Define | /spec | 确定要构建什么 | 2-3个技能 |
| Plan | /plan | 分解工作 | 1-2个技能 |
| Build | /build | 垂直切片实现 | 3-4个技能 |
| Verify | /test | 证明它工作 | 1-2个技能 |
| Review | /review | 捕获遗漏 | 1-2个技能 |
| Ship | /ship | 安全部署 | 1-2个技能 |
| Simplify | /code-simplify | 跨阶段优化 | 1个技能 |

复杂功能可能用11个技能；小bug修复可能用3个。

## 如何在Hermes中使用

### 方法1：直接创建为技能
每个SDLC阶段作为一个独立的Hermes技能，通过触发短语激活。

### 方法2：单技能内嵌步骤
在一个Hermes技能中包含完整的步骤序列+反合理化表。

### 示例：简化版反合理化技能模板

```markdown
---
name: 写spec前的自我检查
tags: [spec, anti-rationalization]
---

## 反合理化表（执行前必读）

| 借口 | 反驳 |
|------|------|
| "这个修改很小" | 越小越应该写清楚，省得review时猜。 |
| "用户不会注意到" | 用户总是会注意到。 |
| "时间来不及" | 不写spec的时间都花在了debug上。 |

## 步骤
1. 写一句话描述这个改动要解决什么
2. 列出3个验收条件
3. 确认不碰的边界
```

## 注意事项
⚠️ 反合理化表需要针对性设计——通用的"用户不会注意到"适用面窄
⚠️ 技能数量不能太多（>20个）会导致Agent选择困难
⚠️ Progressive Disclosure需要元技能做路由——没有路由器的技能集合不如不用
⚠️ 每条技能要有明确的触发性短语，才能在正确时机被Hermes加载
