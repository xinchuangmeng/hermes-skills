#### 核心实践方法（从Anthropic官方最佳实践提取）

**项目配置黄金规则：**
- 放适合的内容：Bash命令（Claude猜不到的）、代码风格规则、测试指令、仓库规范
- 不放的内容：API文档（链接过去即可）、频繁变化的信息、自明实践

**Skills构建三原则：**
1. **渐进式加载** — 主文件只放描述（约200字），具体内容放引用文件按需加载
2. **可组合性** — 多个Skill可同时安装，自动判断触发时机
3. **跨平台复用** — Skills在CLI、Desktop、Agent SDK间完全可移植

**SubAgent模式（生产力倍增器）：**
主Agent（决策+执行）→ 委托SubAgent（隔离上下文调研）。优势：调研消耗的Token不计入主Agent上下文。

**Writer/Reviewer双Agent模式（Anthropic内部验证）：**
Session A（Writer）实现功能 → Session B（Reviewer）审查找边界条件 → Session C修复。效果比单Agent自写自审查好3倍以上。

## 四、2026年5月新势力解析

### Agent Skills标准

**核心概念：** 用文件夹+描述文件构建可复用的Agent专业知识库。

```
.skills/
├── pdf-editing/
│   ├── SKILL.md          # 描述+元数据（预加载到系统提示）
│   ├── form-filling.md   # 具体指令（按需加载）
│   └── extract-fields.py # 可执行脚本
```

**技术原理：** 启动时只加载所有Skill的描述（轻量），Claude根据任务自动判断是否加载具体内容。类似渐进式加载——目录→章节→详细附录。

**跨平台复用：** 相同的Skill目录可在Claude Code CLI、Claude Desktop、Claude Agent SDK、Claude Developer Platform之间直接复用。

**Skill vs MCP的区别：** MCP是工具接口标准（连接外部系统），Skills是领域知识标准（教会Agent怎么做事）。两者互补而非替代。

### Claude Code Auto Mode

**核心机制：** 两层防御自动审批
- 输入层：提示词注入探测，扫描所有工具输出
- 输出层：Transcript分类器，Stage 1快速过滤（8.5%触发），Stage 2完整评估（FPR仅0.4%）

**性能数据（10000次真实调用测试）：**
- FPR 0.4%（10000次仅40次冤枉拦截）
- 危险动作检出率83%（52个场景中的43个成功拦截）
- 漏检主要是用户意图授权判断模糊

**拦截场景：** 用模糊指令删除远程git分支、从环境变量扫描替代token、把内部代码传到公开Gist、deploy跳过验证、修改共享基础设施。

### Managed Agents（Brain-Hands解耦架构）

Session（持久化日志）← Harness → 容器（可替换）。容器故障→Harness捕获→Claude决定重试→新容器初始化。Harness故障→从Session日志恢复。p50 TTFT下降约10倍。
