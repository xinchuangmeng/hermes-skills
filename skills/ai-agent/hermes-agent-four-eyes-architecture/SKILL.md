# Hermes Agent 四眼架构

> 来源：敬哥分享的4张PPT（2026/5/15），讲述Hermes Agent的四大感知工具分层模型。
> 核心思想：Agent的"眼睛"按能力和权限分层，从公开到私密、从数据到操作，逐层递进。

## 一、架构全景

```
┌─────────────────────────────────────────────┐
│           ④ 真实浏览器眼 (bb-browser)        │
│     开真实Chrome,像人一样操控网页/登录/交互    │
├─────────────────────────────────────────────┤
│           ③ 技能集成眼 (Agent-Reach)          │
│   把深度内容/专业知识打包成技能,Agent直接调用   │
├─────────────────────────────────────────────┤
│           ② 平台内容眼 (AutoCLI)              │
│   复用Chrome登录态,抓需要登录的私密平台内容     │
├─────────────────────────────────────────────┤
│           ① 全网搜索眼 (web_search+extract)   │
│   自主搜索公开网页并提取内容,第一只眼睛         │
└─────────────────────────────────────────────┘
```

## 二、每只眼睛详解

### ① 全网搜索眼 —— web_search + web_extract

- **编号**：1（第3页PPT）
- **定位**：Hermes的第一只眼睛，最基础的信息获取能力
- **能力**：
  - `web_search` — 像人一样用搜索引擎（Google/Bing）
  - `web_extract` — 抓取网页正文，过滤广告和杂讯
- **流程**：搜索 → 提取 → 整合，全流程自动完成
- **本质**：给AI装上搜索引擎，让它自己找信息
- **我当前状态**：✅ 每天都在用，回答前必先搜索查证

### ② 平台内容眼 —— AutoCLI

- **编号**：2（第4页PPT）
- **定位**：能看需要登录才能访问的"内部内容"
- **原理**：复用你的Chrome登录态（Cookie/Session），不需要重新登录
- **支持平台**：
  - 微信公众号 — 文章
  - 知乎 — 问答、专栏
  - 小红书 — 笔记、评论
  - 即刻 / 掘金 — 社区内容
  - 飞书文档 — 在线文档
  - 更多平台持续扩展
- **核心优势**：你的登录态它都能用，不需要重新登录就能看内部内容
- **我当前状态**：❌ 未实现。但已有 `youtube-content` 技能做YouTube字幕提取（类似能力）

### ③ 技能集成眼 —— Agent-Reach

- **编号**：3（第5页PPT）
- **定位**：把各种难抓的深度内容打包成技能，Agent直接调用
- **四大模块**：
  1. **YouTube字幕提取** — 获取视频字幕，AI可以"看"视频内容，无需下载视频
  2. **GitHub仓库分析** — 读取README、代码结构、Issue，理解开源项目
  3. **RSS信息流订阅** — 持续追踪博客、新闻源更新，聚合信息源
  4. **更多技能包...** — 支持自定义技能扩展
- **本质**：技能 = 封装好的专业知识/领域能力，Agent按需加载，不需要每次都从头搜索
- **我当前状态**：✅ **核心能力**！我现在有20+个技能，铁纪律要求"学到知识立刻存为技能"

### ④ 真实浏览器眼 —— bb-browser

- **编号**：4（第6页PPT）
- **定位**：不是模拟请求，而是打开真实浏览器
- **与其他工具的区别**：
  - 其他工具：调API、发HTTP请求
  - bb-browser：打开Chrome，看到完整的网页界面
- **能力**：需要登录、需要交互的网站，它都能自己搞定
- **本质**：Agent可以像人一样操控网页
- **我当前状态**：❌ 未实现。Hermes v0.13没有Computer Use能力

## 三、四眼的配合关系

| 场景 | 使用哪只眼 | 
|------|-----------|
| 查公开资料/百科/新闻 | ① 全网搜索眼 |
| 读微信公众号/知乎专栏/飞书文档 | ② 平台内容眼 |
| 分析YouTube视频/GitHub项目/持续跟踪 | ③ 技能集成眼 |
| 操作需要登录的网站/做复杂交互 | ④ 真实浏览器眼 |

**递进关系**：
1. 先用全网搜索眼做广度调研
2. 用平台内容眼深入私密内容
3. 把有价值的知识固化为技能（技能集成眼）
4. 最后用浏览器眼执行操作

## 四、对我（Hermes书童）的启示

1. **已经做对的**：回答前先搜索（全网搜索眼）、学到知识存技能（技能集成眼）
2. **可以改进的**：技能集成眼还可以扩展更多模块（RSS订阅、GitHub分析等）
3. **未来方向**：如果Hermes支持browser能力，可以补全平台内容眼+真实浏览器眼
