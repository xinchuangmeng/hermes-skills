# 图文类短视频全流程——知识卡片+图文成片组合打法

## 核心策略定位

**适用场景：** 面向小白的知识教学类账号，不露脸、不复杂剪辑、靠干货深度建立信任。

**核心打法：** 知识干货号(深度内容) + 图文成片(降低观看门槛) 组合使用

**不适合：** 纯AI图文号（信息太浅）、带货号（需要产品展示）

### ⚠️ 纯图文模式（无口播）—— 用户明确要求

**核心区别：不做口播配音。卡片上的文字就是文案，观众自己阅读。**

| 维度 | 有口播模式（旧） | 纯图文模式（当前） |
|------|:---------------:|:-----------------:|
| 配音 | 需要选配音音色 | **不选配音，BGM是唯一音频** |
| 文案 | 口播文案独立存在 | **文案写在卡片上，无单独口播文案** |
| 字幕 | 配音自动转字幕 | 剪映自动识别卡片上的文字生成字幕 |
| 时长 | 30-60秒 | 55-60秒（每张卡8-10秒，给足阅读时间）|
| BGM音量 | 15-20% | 20-25%（无配音遮挡，音量可大些）|
| 输出物 | 文案+分镜+卡片+音效指引 | **仅出3件套（见下方）** |

**输出物规范（仅3件套）：**

| # | 输出物 | 说明 |
|---|--------|------|
| 1 | **知识卡片设计方案**（每张的文案+配色+图标+设计要点） | 卡片上的文字就是文案本身，观众自己看 |
| 2 | **剪映图文成片制作指引**（BGM/时长/导出设置） | 用户按步骤操作即可 |
| 3 | **系列化选题框架**（如需） | 每期只讲1个"怎么办" |

**不需要输出：** 口播文案、配音稿、运镜描述、灯光体系、Seedance提示词、分镜脚本表、音效设计。纯图文短视频不需要这些。

**每张卡片设计要点（纯图文模式特有）：**
- 文字要自包含，观众看字就能懂全部内容，不依赖声音
- 每张卡只讲1个核心概念，不堆砌
- 字号要大（手机能看清），深底白字最好
- 加图标/箭头/数字等视觉元素辅助理解
- 每张卡8-10秒展示时间，给足阅读缓冲

## ⚠️ 输出物规范（重要）
图文成片技能的输出物限定为**3件套**，不需要电影级运镜/灯光/音效那套：

| # | 输出物 | 说明 |
|---|--------|------|
| 1 | **知识卡片设计方案**（每张的文案+配色+图标+设计要点） | 卡片上的文字就是文案本身，观众自己看 |
| 2 | **剪映图文成片制作指引**（BGM/时长/导出设置） | 用户按步骤操作即可 |
| 3 | **系列化选题框架**（如需） | 每期只讲1个"怎么办"，每期嵌入AI工具场景 |

**关键原则——纯图文模式（无口播）：**
- 用户明确不做口播配音。卡片上的文字就是文案，观众自己阅读
- 不需要额外写口播文案（配音文案）
- 剪映图文成片时：不选配音音色，BGM是唯一音频
- 字幕来源：剪映自动识别图片上的文字生成
- 卡片的文字要自包含，观众看字就能懂全部内容

**不要输出：** 口播文案、运镜描述、灯光体系、Seedance提示词、分镜脚本表、音效设计。纯图文短视频不需要这些。

## 💡 差异化设计铁律：每期嵌入AI工具场景
市面上「新手从0到1」内容泛滥，我们的核心差异是**每期教一个AI工具怎么用**。
- 第1期（纯扫盲）可以不嵌AI
- **从第2期开始，每期必须嵌入至少1个AI工具应用场景**
- 示例：AI写标题 / AI做产品图 / AI分析竞品 / AI算利润 / AI写客服回评
- 每期文案中自然植入「用AI 5分钟搞定」的感知，让小白觉得门槛低

采用"每期只讲1个怎么办"的系列化结构，用户看完一期就知道下一期长什么样：

### 统一卡片结构（6张）：

卡片5：干货③ → 第三个要点

## 内容流水线（5步法）

### Step 1：选题

选题标准：**小白最关心的"怎么办"问题**

- 必须是实操问题（不是理论问题）
- 有明确答案和步骤
- 60秒能讲清楚

示例选题：
- 《Shopee新店前7天做什么》
- 《1688怎么找源头工厂》
- 《Lazada定价公式怎么算》
- 《开店注册总被拒？这3个坑别踩》

### Step 2：写干货文案

**文案结构模板（800-2000字）：**

```
标题：冲击力强，含具体数字（例：《开店第一个月我只做这5件事》）

开头(100字)：痛点钩子
- "你是不是也遇到XXX问题？"
- "我当初开店第一个月完全懵了"

正文：按序号展开每个要点
- 每个要点1-2段，配1张知识卡片
- 每点必须有数据/可执行步骤
- "第一，XXX"
- "第二，XXX"

结尾(50字)：金句总结 + 引导关注
```

### Step 3：做知识卡片（6-9张）

**卡片分配方案（以6张为例）：**

| 编号 | 卡片内容 | 说明 |
|:----:|---------|------|
| 1 | **封面图** | 悬念标题 + 账号标识 |
| 2 | **核心问题** | 痛点展示 + 一句话引出 |
| 3 | **要点1** | 第一个实操步骤/方法 |
| 4 | **要点2** | 第二个实操步骤/方法 |
| 5 | **要点3** | 第三个实操步骤/方法 |
| 6 | **总结金句** | 核心一句话 + 行动号召 |

**卡片的制作要点：**
- 每张卡只讲1个核心概念，不堆砌
- 字号要大（手机能看清）
- 颜色对比鲜明（深底白字最好）
- 加图标/箭头/数字等视觉元素
- 用即梦AI/Canva制作，统一风格

### Step 4：剪映图文成片

**⚠️ 纯图文模式（无口播）操作步骤：**
1. 打开剪映 → 点击「图文成片」
2. 选择6-9张卡片图片（按顺序排好）
3. **不要选配音音色**（用户明确不做口播配音），直接跳下一步
4. 选择BGM（轻知识类，音量20-25%，因为BGM是唯一音频）
5. 点击「一键生成」
6. 微调：
   - **字幕检查**：剪映会自动识别图片上的文字生成字幕，快速扫一眼是否有错别字
   - **图片时长**：每张图调为8-10秒（比有口播的5-8秒略长，给阅读时间）
   - 总时长控制在55-60秒
7. 导出：竖版9:16，分辨率1080p

**关键注意事项：**
- 卡片上的文字就是文案本身，观众自己阅读，不需要口播稿
- 剪映会自动识别图片上的文字生成字幕，导出前扫一眼错别字
- BGM是唯一音频，音量比有口播时可以大一些（20-25%）
- 不要在剪映里额外加配音音色

**时长控制：**
- 6张图 × 每张8-10秒 ≈ 48-60秒
- 总时长控制在60秒以内，完播率最高

### Step 5：多平台分发

| 平台 | 内容形式 | 说明 |
|------|---------|------|
| 视频号 | 图文成片视频 | 主阵地，每天1条 |
| 公众号 | 完整干货文章 | 1500-2500字深度版 |
| 头条号 | 精简版文章 | 600-1000字 |

## 📐 卡片风格选型：参考抖音真实爆款风格

**关键发现：不要自己设计风格，参考抖音上点赞高的图文干货账号的风格。**

通过分析3组抖音图文视频（共282MB录屏素材），总结出3种主流风格：

### 风格①：米黄笔记风（推荐 ✅）
**代表账号：** @主播控场小技巧（18W+赞）
**适合场景：** 知识干货类、教程类、清单类
**特点：**
- 背景：米黄/奶油色纯底（类似便签纸/护眼模式）
- 文字：黑色无衬线黑体，左对齐编号列表
- 层级：三级字号（标题大红/深棕 → 副标题灰黑 → 正文黑）
- 无任何图片/插画/特效，纯文字干货
- **制作超简单，剪映图文成片直接配字就行**
- **推荐理由：信息密度高、制作门槛低、适合小白向教学**

### 风格②：复古警示牌风（偶尔用）
**代表账号：** @涛哥新媒体
**适合场景：** 揭露内幕、避坑指南、警示类内容
**特点：**
- 做旧米白底 + 深红警示条 + 红白斜条纹（施工牌风格）
- 插画+文字混排，视觉冲击强
- 比风格①复杂，需Canva/PS制作

### 风格③：暗黑科技风（不推荐）
**代表账号：** @吉少电商日记
**适合场景：** 干货揭秘、利益点突出
**特点：**
- 纯黑底 + 星光粒子 + 金色光束
- 金色发光立体大字
- 电影感，制作门槛高
- **不推荐原因：与小白向教学定位不符，制作成本高**

### 选型建议
- **系列化干货内容（新手从0到1）→ 风格①**
- 偶尔的避坑/警示类 → 风格②
- 不要用风格③（太复杂，而且我们之前生成的暗黑风卡片被用户拒了就是因为这个）

## 🛠 实操经验：Pillow生成卡片时的常见坑

1. **TTC字体索引问题**：`NotoSansCJK-Bold.ttc` 是TTC合集，Pillow加载时默认取 `index=0`，对应Regular。想用Bold/Black字重必须指定 `index`：
- `index=0` → Regular
- `index=1` → Bold （对 `NotoSansCJK-Bold.ttc` 而言）
- 验证方法：分别跑 `ImageFont.truetype(path, size, index=0)` 和 `index=1`，对比效果
- 不同TTC文件index映射不同！最好先测试一下

2. **中文换行**：Pillow的 `text()` 不支持自动换行。一行超过12-15个字建议手动拆成两行分别绘制，保证手机屏幕可读。左对齐文字建议用 `anchor="lm"`，居中用 `anchor="mm"`。

3. **emoji兼容性**：Noto CJK字体不支持emoji渲染，用emoji字符（💰⏰📦✅❌）会显示为方框。**解决方案**：用文字描述代替emoji（如「💰钱」），或改用纯文字符号（👉 ★ ✓ ✗）。不要用emoji做核心视觉元素。**解决方案**：用文字描述代替emoji（如「💰钱」），或者换用支持emoji的字体（如Noto Color Emoji），但颜色控制会变复杂。最简单：不用emoji，改用纯文字符号（👉 ★ ✓ ✗）。

4. **输出目录权限**：OUTPUT_DIR用相对路径时，确保运行目录有写入权限。建议用绝对路径。

5. **视觉验证**：生成后建议用 `vision_analyze` 检查每张卡片的效果——文字清晰度、排版是否拥挤、配色是否协调。至少检查封面、第一张内容卡和结尾卡三张。

6. **快速迭代技巧**：每张卡写一个独立函数 `card1(draw, img)` 到 `card6(draw, img)`，改文案时只改 `draw.text()` 里的字符串和坐标。下期复制整个文件再改，不要复用同一个文件。

5. **快速迭代技巧**：每张卡写一个独立函数，改文案时只改 `draw.text()` 里的字符串，不需要动结构代码。这期改完的文字，下期复制文件再改，不要复用同一个文件。

## 避坑要点

1. **不要纯文字截图**——小白看着累，必须配图/图标
2. **不要一次讲太多**——每期1个问题就够了，别贪多
3. **每张卡留白足够**——手机屏幕小，字太多看不清
4. **封面图决定打开率**——标题要冲击力，数字+反常识
5. **图文成片≠偷懒**——图片质量决定观看体验，要花心思设计

## 七、服务端自动生成知识卡片图片（Python Pillow方案）

当用户需要快速出图、不想在Canva/PPT里手动做卡片时，可以用Python在服务器上直接生成6张知识卡片图片（竖屏9:16，1080×1920）。

### 前置条件
```bash
pip install Pillow  # 通常已预装
# 需要中文字体（Noto CJK）
# 检查：fc-list :lang=zh | head -3
```

### 核心模板脚本（Pillow画卡片）
参考文件：`~/sea-ecommerce/series/系列一_新手开店从0到1/gen_cards.py`

**脚本结构（每个卡片一个独立函数）：**

```python
from PIL import Image, ImageDraw, ImageFont

W, H = 1080, 1920
BG_COLOR = "#0a0a0a"  # 近黑色
FONT_BOLD = "/usr/share/fonts/opentype/noto/NotoSansCJK-Bold.ttc"
FONT_REGULAR = "/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc"
FONT_LIGHT = "/usr/share/fonts/opentype/noto/NotoSansCJK-Light.ttc"

def create_card(filename, draw_func):
    img = Image.new("RGB", (W, H), BG_COLOR)
    draw = ImageDraw.Draw(img)
    draw_func(draw, img)
    os.makedirs(OUTPUT_DIR, exist_ok=True)
    img.save(os.path.join(OUTPUT_DIR, filename), "PNG")
```

### 6张卡片标准模板函数

每张卡用一个 `cardX(draw, img)` 函数实现，可自由替换文案和配色：

| 卡片 | 编号 | 配色方案 | 功能 |
|:----:|:----:|:--------:|:----|
| cover | 封面 | 黑白灰 | 悬念标题+引导语 |
| pain | 痛点 | 黑+红白 | 痛点场景+反问 |
| point1-3 | ①/②/③ | 蓝/绿/红 | 核心干货要点 |
| end | 结尾 | 黑+金字 | 总结+预告+品牌 |

**配色方案参考：**
- ①预算：深蓝底 + 白色字 + 金色¥和数字高亮
- ②平台：深绿底 + 白色字 + 绿色✅ / 红色❌
- ③时间：暗红底 + 白色字 + 红色警告
- 结尾：纯黑底 + 金色预告

### 不用改脚本，只改文案和配色
模板脚本中的每张卡片文字、颜色、字号都写死在 `cardX()` 函数里。做新一期时：
1. 复制 `gen_cards.py` 为新文件
2. 修改每张卡的文字内容
3. 运行 `python3 gen_cards_v2.py`
4. 从输出目录取出6张PNG
5. 给用户：导入剪映→图文成片→完成

### 注意
- 中文字体路径在不同系统可能不同，先用 `fc-list :lang=zh` 确认
- 字体文件TTC中提取单字重需指定 `index=0`（常规）或 `index=1`（粗体）
- 文字锚点用 `anchor="mm"`（居中对齐）或 `anchor="lm"`（左中对齐）
- 数字和货币符号最好单独设置字体（Bold），突出视觉层次
