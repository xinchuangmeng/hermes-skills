# Shopee虾皮大学官方规则OCR提取入库工作流

## ⚠️ 前提条件

- **小书童（腾讯云服务器）** 无法访问虾皮大学——页面是SPA（JS渲染），curl只能拿到空白加载页
- 腾讯云IP可能已被Shopee风控封禁
- **必须由小强（OpenClaw，Windows本地）或旺财（Hermes微软版，Windows本地）** 执行截图操作
- 截图前加载 `shopee-browse-anti-bot` 技能，真人节奏操作，避免触发风控

## 完整工作流

```
敬哥梳理「还缺什么」→ 小书童出任务清单
  → 敬哥私聊转发给小强/旺财
    → 小强/旺财截图
      → 截图发给小书童
        → 小书童OCR提取文字
          → 更新对应skill
```

## 一、任务梳理阶段（小书童执行）

每次需要从虾皮大学获取数据时：

1. 先列出 **当前skill中已有数据**（哪些已确认）
2. 再列出 **还缺的数据项**（交易手续费率？SLS运费单价？平台基础设施费？）
3. 针对每个缺失项，明确 **需要截什么页面、截什么位置**
4. 整理成 **任务清单**，发给敬哥转小强

### 输出格式

```
## 任务：需要的截图清单

### 1️⃣ [页面名] → [路径]
需要截的内容：
- [具体数据1]
- [具体数据2]
- 截清晰大图，包含百分比数字

### 2️⃣ [页面名] → [路径]
...
```

## 二、执行阶段（小强/旺财执行）

### 2.1 加载真人化浏览技能
```
加载 shopee-browse-anti-bot 技能
→ 按以下节奏操作：
  - 新页面停留8-15秒
  - 慢速滚动浏览
  - 每3个页面停顿20-40秒
  - 00:00-06:00不操作
```

### 2.2 截图要求
- **清晰大图**，不要裁剪边缘
- 包含完整的数字、表格、百分比
- 如果是一个页面有多屏内容的，分段截
- 每张图命名对应页面名（例如 `commission-fee-table.jpg`）

### 2.3 已知可以爬取的页面
| 页面 | 路径 | 已成功获取 |
|:----|:----|:---------|
| 新手须知→佣金/服务费/交易手续费 | shopee.cn/edu 搜索"佣金" | ✅ 提取到佣金18.63%等 |
| 物流→SLS跨境物流手册/藏价 | shopee.cn/edu 搜索"藏价" | ✅ 提取到藏价机制说明 |
| 泰国站专区→税务/VAT | shopee.cn/edu 泰国站专区 | ❌ 未提取到VAT税率数字 |
| 卖家中心→公告栏 | seller.shopee.cn → 顶部公告 | ❌ 未获取 |
| SLS运费计算器 | 企业微信扫码下载Excel | ❌ 无法截图 |

### 2.4 注意事项
- 虾皮大学页面内容形式：**图片+PPT模式**（不是纯文字HTML）
- 所有内容都是图片嵌入，必须截图
- 不要复制粘贴文字（复制不了）

## 三、OCR处理阶段（小书童执行）

### 3.1 OCR工具选择

| 工具 | 中文识别 | 表格识别 | 速度 | 推荐度 |
|:----|:--------|:--------|:----|:------|
| **Tesseract 5** (chi_sim+eng) | ⭐⭐⭐ | ⭐ | 快 | 首选 |
| **KIMI API** (moonshot-v1-32k) | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 慢（需base64） | 辅助验证 |
| **DeepSeek视觉** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 快 | 当前环境不可用 |

### 3.2 Tesseract命令
```bash
# 单张图片
tesseract /path/to/image.jpg /tmp/ocr_output -l chi_sim+eng --psm 6

# 批量处理
for img_id in id1 id2 id3; do
    tesseract "/root/.hermes/image_cache/img_${img_id}.jpg" "/tmp/ocr_${img_id}" -l chi_sim+eng --psm 6
done
```

### 3.3 Tesseract已知限制
- 中文识别率约60-70%（图片中的中文）
- **表格数字**识别效果更差（约40-50%）
- 关键百分比数字容易乱码或丢失
- 解决方法：先用Tesseract提取全部文字，再用DeepSeek/KIMI做**数据整理和数字抽取**

### 3.4 AI辅助数据整理
```python
# 把OCR原始输出 + 上下文描述 → 发给KIMI/DeepSeek提取关键数字
prompt = f"""
从以下OCR提取的Shopee虾皮大学页面文字中，
提取泰国站的核心费率数据和规则。
只提取OCR中明确出现的具体数字和条款，不要编造。

原始OCR数据：
{ocr_text}

问题：
1. 泰国站SLS渠道的佣金费率是多少？
2. 交易手续费率是多少？
3. 服务费率是多少？哪些场景收？
4. 免佣规则是什么？
"""
```

### 3.5 OCR后处理
1. 提取到关键数字后，**标注来源**（虾皮大学 + 更新日期）
2. 对比skill中现有数据，标记哪些被修正、哪些是新增
3. 如果OCR有歧义或数字不清楚，标记为**待确认**，记录在任务清单下次补截图

## 六、已知局限与改进方向

| 局限 | 原因 | 改进方案 |
|:----|:----|:--------|
| 表格数字OCR准确率低 | Tesseract中文表格识别差 | 截图时放大表格区域；用KIMI辅助识别 |
| 无法直接抓取 | SPA+JS渲染 | 只能靠截图，无法自动化 |
| 小书童不能用 | 服务器IP被封 | 必须走小强/旺财本地浏览器 |
| 飞书群消息无人响应 | 小强/旺财不是飞书Bot | 敬哥私聊转发 |
