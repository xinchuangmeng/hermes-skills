# 电影级短视频脚本生成系统

## 一、爆款文案3层结构

### 第1层：黄金3秒钩子（强行留住观众）
```
[反常识提问] "你知道吗，XX其实不是你想的那样……"
[数字冲击] "月销3000+的店，差评200条，但我看到了2000个机会"
[身份转换] "新手开店第一个月，我做了这5件事，少一件白忙三个月"
[痛点直击] "你是不是也这样：每天花3小时选品、写文案、做图，累得要死……"
```

### 第2层：干货内容节奏（每8-12秒一个信息点）
- **8-12秒节奏**：人的注意力15秒上限，每12秒必须给一个新信息或视觉刺激
- **3段式结构**：抛出问题 → 拆解分析 → 给解决方案
- **数据锚点**：每个观点必须有具体数字（"每天30块""月销3000+""200条差评"）
- **场景切换**：每12秒画面必须变化（特写→中景→切换场景）

### 第3层：尾钩+互动引导
```
[价值承诺] "想拿完整版操作清单？私信回复'清单'免费发你"
[下期预告] "下期教你用AI 5分钟扒光同行数据"
[悬念留白] "至于怎么用AI做竞品分析——下期拆给你看"
```

## 十、BGM/音效选择指南

### 知识干货类
- **开场**：弱起节奏、低沉氛围（音量15%）
- **主体**：轻快知识科普风（音量20%）
- **高潮**：节奏感稍强（音量18%）
- **结尾**：渐弱淡出

### 故事类
- **开场**：悬念氛围（音量15%）
- **发展中**：轻叙事（音量20%）
- **转折**：节奏加强（音量22%）
- **高潮**：情绪推升（音量20%）
- **结尾**：温馨/悬念收尾（音量15%）

### 音效触发点
- **关键数字出现**：标记音「叮」或「叮叮叮」
- **数据展示**：翻页声/打字声/扫描声
- **场景切换**：过渡音效「咻」
- **悬念收尾**：低频「嗡……」
- **转场高潮**：短促鼓点「咚」

## 十一、文案写作铁律

1. **每8-12秒一个信息点**，不能超过12秒不换画面
2. **每句话必须有一个具体数字**（数量/金额/百分比/时间）
3. **少用形容词，多用动词**（"搞""扒""干""冲"比"优秀""精彩""强大"有力10倍）
4. **全程用"你"做主语**，让观众感觉在跟他说话
5. **转折处压低声调放慢速度**，制造悬念感
6. **结尾留尾钩，不要引导关注**，而是给价值（免费私信领资料）
7. **每3个视频一个系列化**，连续剧式尾钩承接

## 十二、致命陷阱：画面时长必须≥配音时长

**❌ 错误做法：** 生成5段5秒画面（共25秒），但配音60秒→画面循环2-3遍→观众一眼看出重复→划走

**✅ 正确做法：**
- 60秒配音需要**至少6-7个独特场景**（每个8-12秒），让画面总时长≈配音时长
- 60秒配音 → 每个场景10秒 → 6个场景
- 75秒配音 → 每个场景12秒 → 6-7个场景（含封面3秒）
- 如果API只能生成5秒片段，那就多生成几段：12段×5秒=60秒

**场景数量速算表：**
| 视频时长 | 场景数量（每段5秒） | 场景数量（每段10秒） |
|---------|------------------|------------------|
| 45秒 | 9段 | 5段 |
| 60秒 | 12段 | 6段 |
| 75秒 | 15段 | 7-8段 |

**总结：宁可画面多到用不完，也不要画面不够导致循环。**

## 十三、⚠️ 生产坑点（实际踩坑记录）

### 配音生成（SiliconFlow CosyVoice2）
1. **录音样本不能超过30秒**（含尾音），否则报 `audio longer than 30s is not supported`。截取29秒最安全。
2. **上传声音样本要用multipart/form-data**，提供`text`字段（样本对应的文字内容）和`file`字段（wav格式，16kHz，单声道）。
3. **动态声音URI格式**：`speech:{customName}:{userId}:{token}`，在TTS请求的`voice`字段中直接使用。
4. **短文本可能返回空文件**（HTTP 200但0字节），切成10-30字短句时偶发。建议每段不少于30字。
5. **先预设声音测试**：用 `FunAudioLLM/CosyVoice2-0.5B:alex` 测试API通不通，再切到动态声音。
6. **余额不足才报错**（code 30001），不是API坏了。去 https://cloud.siliconflow.cn/ 充值最低1元起。

### Seedance视频生成（火山引擎方舟）
1. **API Key格式**：`ark-xxx`，与AK/SK（AccessKey/SecretKey）是两套鉴权。不要混淆。
2. **模型需要先开通**：去方舟控制台 → 视觉模型 → 点"开通"。1.5-pro有200万免费tokens。
3. **生成任务异步**：提交后返回task_id，轮询5秒/次直到 status=succeeded。
4. **视频URL 24小时过期**，需要及时下载。
5. **画面带自动生成音频**（generate_audio: true），合成时注意音频轨道叠加。
6. **横屏21:9比例**，不适合竖版视频。需在FFmpeg合成时pad到1080×1920。

### FFmpeg合成
1. **配音长于画面**：用 `-stream_loop -1` 循环视频画面，`-shortest` 以最短流（配音）为结束。
2. **WAV片段合并**：用concat demuxer，先写file list.txt再ffmpeg -f concat合并。
3. **封面转视频**：`-loop 1 -i cover.png -t 3 -c:v libx264` 把静态图转成3秒视频片段。
4. **画面循环编码慢**：`-c:v copy`（不重新编码）远快于 `-c:v libx264`，优先用copy。

### 文件传递
1. **scp下载视频到Windows**：在cmd执行 `scp agentuser@IP:远程路径 本地路径`
2. **飞书API上传文件**：需tenant_access_token + 上传到im/v1/files + 发送file消息到群聊
3. **腾讯云安全组策略**：外网端口默认全关，需控制台放行才能访问

## 附：首帧与尾帧设计体系

### 核心原则

**首帧质量 = 整个视频质量的下限。** 首帧决定了角色身份、光影、色调。首帧有瑕疵，后面每一帧的误差都会累积放大。

**首帧+尾帧之间的差异 = 你讲的那个故事。** 两张图之间的空间就是故事，AI是你的叙述者。

### 账号定位决定首尾帧策略

以「我和AI做跨境电商」为例（AI智能体+跨境电商，悬念故事型叙事）：

| 元素 | 策略 |
|------|------|
| **首帧** | **每期不同，但视觉家族感统一** |
| **品牌尾板**（片尾落版） | **固定模板，每期复用** |
| **收尾叙事画面** | **每期不同，但构图风格统一** |
| **引导话术** | 固定句式「想拿XX？私信回复XX免费发你」 |
| **尾钩预告** | 每期不同 |

### 首帧设计铁律

1. **深色底+冲击型开场+3秒内亮核心信息** — 每期如此
2. **开场的具体画面和运镜方式** — 根据当期内容变化
3. **3秒无配音留白** — 固定技巧，制造悬念
4. **首帧宽高比必须匹配输出视频宽高比**
5. **首帧留出运动空间** — 主体不填满画面，留10-15%边距

### 尾帧设计铁律

1. **保持同一场景和环境** — 不要换场景
2. **保持角色身份** — 同一个人，同样衣服同样发型
3. **只改变应该改变的** — 位置/姿势、表情、时间推移
4. **微妙变化产出更平滑结果** — 小幅转头比剧烈动作成功率高10倍
5. **剧烈变化需要更长时长** — 3秒仓促→10-15秒自然
6. **差的尾帧比没有更糟** — 没有合适尾帧时宁可不用

```

第0.5秒：金色图标淡入（箭头/三角形，代表方向感）
```

## 十七、社区精华：Seedance提示词进阶技巧（2026.5月爬取）

### 技巧1：权重顺序
```
参考图 > 主体 > 动作 > 光影 > 镜头 > 画质 > 稳定约束 > 负面词
```
参考图（如果有）的优先级最高，紧接着是主体定义，最不重要的是负面词。

### 技巧2：镜头唯一原则
**单条提示词只写1种运镜**，多镜头必闪屏/割裂。
- ❌ 错误：先推近再环绕再拉远（一个提示词里写3种运镜）
- ✅ 正确：一段画面=1个镜头，多个镜头用多个提示词生成后拼接

### 技巧3：Seedance 2.0的@语法
```
@图片1 作为首帧，定下整体画风
参考@视频1的运镜方式，从全景推到特写
@图片1 中的女生作为主角，保持服装和脸部一致
@音频1 作为配乐，对齐画面节奏
```
**Seedance 2.0核心玩法**：用@语法明确告诉模型每个素材的用途。

### 技巧4：人物一致性保持
```
1. 第一次生成 → 保存最满意的镜头截图
2. 第二次生成 → 将截图作为@参考图上传
3. 提示词明确说：@第一个镜头的女生作为主角，保持服装和脸部一致
```

### 技巧5：多片段无缝衔接
提示词中写3段结构：
```
开始：缓慢推轨推近建立场景
然后：主角进入，动作加快
最后：2秒轻柔向右平移，引出下一幕
```

### 技巧6：Seedance 2.0 vs 1.5 pro 区别
| 能力 | 1.5 pro | 2.0 |
|------|---------|-----|
| 多模态输入 | 文本+图片 | 最多9图+3视频+3音频 |
| 最大时长 | 12秒 | 15秒 |
| @语法 | 不支持 | 支持 |
| 人物一致性 | 一般 | 很好 |
| 自动运镜 | 需要手动描述 | 自动规划 |

### 技巧7：避免的坑
1. **避免剧烈动作**：快速旋转、剧烈抖动容易崩
2. **避免无效描述**：不要写"美丽的""好看的"这种虚词
3. **避免多主体**：一个场景里人物越少越稳定
4. **避免复杂背景**：纯色/虚化背景成功率最高
5. **留白要写清楚**：不说"画面暗下"模型不会自动暗

## 十九、剪映进阶调色（社区精华）

### 基础调色公式
```
锐化: 20
对比度: +5~8     → 去灰
高光: -5         → 不过曝
阴影: +5~10      → 暗部细节保留
色温: -3~-5      → 偏冷显干净
饱和度: +3~5     → 色彩微加
```

### HSL调色（去土气核心）
- 橙色饱和度+10~15：提亮肤色
- 蓝色饱和度+5~10：背景通透
- 绿色饱和度-5~10：去脏感
- 红色饱和度+3~5：唇色自然

### LUT预设推荐
- 青橙色(Tiffany Blue)：干净清爽 → 干货/口播
- 胶片(Film Grain)：温暖复古 → 故事型
- 黑金色：冲击力强 → 数据/对比类
- 强度推荐30-60%，不要100%
