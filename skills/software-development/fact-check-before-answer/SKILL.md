# 先查证再回答协议

## 核心原则

> **回答必须严谨，要经过网上或有确切消息后再回答，避免出现信息差错误。**
> —— 用户明确要求

## 工作流

```
收到用户问题
    ↓
判断是否需要实时信息？
    ↓ 是
用Tavily搜索查证（search + 必要时extract抓取全文）
    ↓
多个来源交叉验证
    ↓
在回答中注明信息出处（"根据XXX的资料"）
    ↓
给出有依据的回答
```

## 什么情况必须搜索

| 场景 | 举例 | 必须搜？ |
|------|------|---------|
| 平台运营策略 | 视频号一天发几条、最佳发布时间 | ✅ 必须 |
| 工具推荐 | 剪映/即梦/可灵哪个好、价格 | ✅ 必须 |
| 时效性信息 | 2025/2026年最新趋势、政策 | ✅ 必须 |
| 具体数据 | 播放量多少、转化率多少 | ✅ 必须 |
| 技术配置 | Hermes怎么配、API怎么用 | ✅ 必须（配官网文档） |
| 常识性知识 | 9:16是什么意思 | ❌ 不需要 |
| 用户自己的项目 | 你的视频数据怎么样 | ❌ 不需要 |

## 搜索策略

### 用Tavily的两种方式

**方式一：先search搜结果（推荐）**
```bash
# 搜索→看摘要→判断是否需要深入
curl -s "https://api.tavily.com/search" \
  -H "Content-Type: application/json" \
  -d '{"api_key":"$TAVILY_API_KEY","query":"你的搜索词","search_depth":"basic","max_results":8}'
```

**方式二：如果摘要不够，用extract抓全文**
```bash
curl -s "https://api.tavily.com/extract" \
  -H "Content-Type: application/json" \
  -d '{"api_key":"$TAVILY_API_KEY","urls":["https://具体链接"]}'
```

### 搜索技巧
- **同一问题用2-3组不同关键词**搜索，交叉验证
- 优先搜行业权威来源（新榜、知乎高赞、官方文档、Reddit高赞）
- 如果有矛盾信息，把矛盾点也告诉用户"不同来源说法不同"
- 中文问题用中文搜，英文问题用英文搜

## 回答模板

```
根据[XXX来源]的资料：

[具体回答]

---

信息来源于：
1️⃣ 来源A（链接）
2️⃣ 来源B（链接）
（如果有矛盾也在下面注明）
```

## 已知陷阱

1. **在这个腾讯云环境**，browser工具访问谷歌/百度会被验证码拦截 → 不要用browser做搜索，用Tavily API
2. **Tavily免费版每月1000次** → 够用但不要浪费，一条问题搜1-2次足够
3. **搜索关键词要精准** → 模糊搜会找到不相关内容
4. **不要只看一条搜索结果** → 除非是非常权威的官方来源，否则多查两三个来源
5. **桌面软件≠可程序化操控** — 不要默认"剪映/PS/Office"这类GUI软件可以被CLI智能体直接操作，要查证它是否有命令行接口或API
6. **WSL2有桌面鸿沟** — 跑在WSL2里的智能体不能操控Windows桌面GUI，这是Linux隔离层的限制，不是配置问题
7. **Claude Code/Codex/Hermes = CLI工具** — 它们本身不能操作桌面GUI程序，需要pyautogui等中间层，且这层非常脆弱
