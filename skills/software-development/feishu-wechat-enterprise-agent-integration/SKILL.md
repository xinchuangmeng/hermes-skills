### 2.2 三种接入方式（2026更新）

#### 方式A：飞书机器人助手（零代码SaaS）- 2026升级
- 入口：飞书 → 机器人助手 → 创建应用
- 新增触发器：AI任务规划自动触发、多维表格数据分析触发
- 新增操作：一键生成PPT、网页发布、跨表数据联动
- 最适合：非技术团队快速实现内部自动化

#### 方式B：飞书开放平台自建应用（API开发）
- 入口：https://open.feishu.cn/app
- 获取 App ID + App Secret → 添加机器人能力 → 订阅事件 → 开通权限 → 开发服务端
- 核心API：发送消息 `POST /open-apis/im/v1/messages`
- 2026新增：飞书CLI插件 - 安装后可在OpenClaw中直接调用飞书CLI执行系统级操作

#### 方式C：OpenClaw接入飞书（2026最新 - 推荐方案）
- 飞书官网一键部署通道：`openclaw.feishu.cn`
- 支持接入：阿里云/腾讯云/火山引擎
- 核心配置项：appId、appSecret、connectionMode（websocket）、dmPolicy、groupPolicy
- 两种身份模式：
  - 机器人身份：飞书群聊中的团队助手，适合问答、通知
  - 你的身份：可以你的身份操作飞书（发消息、读文档、排日程），需授权
- 数据安全：数据不出企业，零配置接入

## 四、微信智能体机器人（14亿最大入口方案）

### 4.1 2026微信三驾马车：ClawBot / QClaw / WorkBuddy

#### ClawBot（官方插件 - 最轻量）
- 发布时间：2026年3月22日
- 入口：微信 → 设置 → 插件 → ClawBot
- 核心逻辑：纯消息通道，不直接掌握系统权限
- 使用方法：扫码或复制命令绑定已部署的OpenClaw实例
- 支持：文本/图片/视频/文件/语音消息互动
- 优势：微信官方背书，零封号风险
- 局限：仅私聊，不支持群聊直接唤醒

#### QClaw（腾讯云 - 个人远程控制）
- 定位：通过微信控制本地电脑的远程AI助手
- 适用场景：个人用户处理本地文件、查询信息、简单操作
- 原理：在本地电脑安装QClaw客户端，绑定微信后通过私聊下达指令
- 优势：免费公测中，零门槛上手
- 局限：能力受限于本地权限+模型能力

#### WorkBuddy（腾讯自研 - 企业自动化）
- 定位：腾讯自研企业级AI自动化助手
- 适用场景：职场自动化、复杂任务编排、多系统打通
- 原理：云端部署，企业付费使用，深度绑定腾讯云生态
- 优势：企业级安全合规，专业自动化能力
- 局限：需要企业付费，自由度不如自部署OpenClaw

**三选一决策**：

```
需求 → 
├─ 个人尝鲜、远程控制自己电脑 → QClaw（最方便）
├─ 已部署OpenClaw，想在微信上用 → ClawBot（零风险）
├─ 企业需要专业AI助手 → WorkBuddy（付费）/ 自部署OpenClaw（自管）
└─ 要最大化AI能力+自定义 → 自部署OpenClaw + ClawBot接入
```

### 4.2 自部署OpenClaw + 微信接入方案

1. **服务器部署OpenClaw**（阿里云/腾讯云一键部署，199元/年起）
2. **安装微信插件**：`openclaw plugin install openclaw-plugin-wechat-json`
3. **绑定ClawBot**：微信 → 设置 → 插件 → ClawBot → 扫码绑定
4. **或直接使用企业微信通道**：通过企微机器人+OpenClaw间接触达微信用户

### 7.2 落地避坑指南（2026实战经验）

1. **不要一开始就追求全功能** → 先选一个核心场景跑通
2. **知识库是刚需** → 没有精准知识库的AI Bot形同虚设
3. **安全权限要前置** → 企业内网系统与IM集成是生产落地的最大门槛
4. **人在回路(Human-in-the-Loop)** → 完全替代不现实，AI初审+人工终审是最佳实践
5. **避免供应商锁定** → 优先OpenClaw/Dify等开源方案，模型层用多模型路由

7. **OpenClaw版本频繁更新** → 建议：稳定版+测试环境，10分钟拉起新实例
8. **多IM接入不要自己开发通道** → OpenClaw官方插件已覆盖主流IM，直接装插件
