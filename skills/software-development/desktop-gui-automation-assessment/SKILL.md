# 桌面GUI软件自动化可行性评估与替代方案设计

## 核心原则

1. **优先自动化，不推给用户** — 用户找智能体就是为了少干活，别轻易说"你来操作"
2. **但自动化质量不能妥协** — 脆弱的自动化（pyautogui固定坐标方案）只适合演示，不适合生产
3. **永远给出可选方案ABC** — 让用户选，不要只有一个方案

## 评估框架

### 第一步：判断软件类型

| 软件类型 | 自动化可行性 | 典型代表 |
|---------|-------------|---------|
| **命令行工具/CLI** | ✅ 高 — 直接终端调用 | FFmpeg, ImageMagick, yt-dlp |
| **有开放API** | ✅ 高 — 调用API即可 | 即梦AI, ChatGPT, 剪映的云端功能 |
| **桌面GUI（无API）** | ❌ 低 — 需要pyautogui模拟操作 | 剪映桌面版, 微信Windows版, PS |
| **网页版** | ⚠️ 中 — 可用Playwright/Selenium | 剪映网页版, 即梦网页版, Canva网页版 |

### 第二步：如果确实是桌面GUI软件

**能直接操控的方案（如实评估）：**

| 方案 | 实现方式 | 可靠性 | 维护成本 |
|------|---------|--------|---------|
| **pyautogui + 坐标点击** | Python模拟鼠标键盘 | ❌ 极低 — 窗口位置变就废 | 高 — 每次软件更新重写 |
| **pyautogui + 图像识别** | 截图→找图→点击 | ⚠️ 中等 — 受分辨率和主题影响 | 中 — 换皮肤/调分辨率就废 |
| **AutoHotkey脚本** | Windows原生自动化脚本 | ⚠️ 中等 — 依赖窗口标题和控件ID | 中 |
| **WinAppDriver/UIAutomation** | Windows UI自动化框架 | ✅ 较高 — 按控件ID定位 | 中 — 控件ID变就废 |

**结论：桌面GUI自动化可行但极其脆弱，除非用户接受"演示级"效果，否则不建议。**

### 第三步：设计替代方案（关键）

**思路：不是让智能体操作软件，而是让智能体产出软件能吃的文件。**

#### 方案A：产出"中间文件"（推荐）

智能体在服务器上生成软件能导入的文件格式，用户只需"下载→导入→套用"：

| 目标软件 | 能产出的文件 | 用户操作量 |
|---------|-------------|-----------|
| **剪映** | 草稿模板(.draft)、LUT调色文件(.cube)、字幕文件(.srt)、封面图(.png) | 下载+导入+套用 ≈ 2分钟 |
| **PR** | 字幕文件(.srt)、LUT(.cube)、XML时间线、封面图 | 下载+导入 ≈ 2分钟 |
| **PS** | 模板PSD、脚本(.jsx)、调色预设、文字图层 | 下载+打开 ≈ 1分钟 |
| **Excel** | 模板文件(.xlsx)、宏(.bas)、公式配置 | 下载+打开 ≈ 1分钟 |

#### 方案B：云端API替代（质量最高）

如果目标功能的云端替代更好：

| 目标 | 桌面软件 | 云端API替代 | 效果对比 |
|------|---------|------------|---------|
| **视频调色** | 剪映调色面板 | FFmpeg色彩处理 + LUT生成 | ✅ 更精确，可批量 |
| **AI视频片段** | 剪映/PR自己找素材 | 即梦API/可灵API生成 | ✅ 更快更专业 |
| **封面设计** | PS/Canva手动做 | Python+Pillow自动生成 | ✅ 可批量，风格统一 |
| **字幕生成** | 剪映智能字幕 | Whisper语音识别+SRT生成 | ✅ 更准确，可批量 |
| **配音生成** | 剪映AI配音 | 魔音工坊/火山TTS API | ⚠️ 各有优劣 |

#### 方案C：用户录制+服务器处理（最省方案）

```
用户录口播（手机/麦克风，1分钟）
    ↓ 传音频文件到服务器
服务器处理（FFmpeg调色+LUT+字幕+即梦片段+BGM合成）
    ↓ 输出成品视频
用户下载 → 直接发视频号（1分钟）
```

这个方案中用户只做2件事：**① 录口播 ② 下载上传**。服务器做全部后期。

### 第四步：给用户展示的格式

用表格清晰地展示3个方案，让用户能一眼对比：

```
## 方案对比

| 方案 | 你做什么 | 我做/服务器做 | 耗时 | 质量 |
|------|---------|--------------|------|------|
| A: 产出文件你套用 | 下载+导入+套用(2分钟) | 生成模板/LUT/封面/字幕 | 即时 | ⭐⭐⭐ |
| B: 云端API全套 | 下载上传(1分钟) | 全部合成，输出成品视频 | 5-10分钟 | ⭐⭐⭐⭐ |
| C: 桌面自动(不推荐) | 等它慢慢点 | pyautogui模拟操作 | 不稳定 | ⭐⭐ |
```

## 已知陷阱

1. **不要轻易说"你操作吧"** — 用户找智能体就是为了少干活
2. **但也不要硬推脆弱方案** — pyautogui固定坐标方案，窗口一挪位就废，用户更生气
3. **剪映草稿模板格式不公开** — .draft格式是剪映内部JSON，版本更新可能变，需要持续维护
4. **即梦API需要用户自己去开通** — 需要用户用抖音/头条账号去jimeng.jianying.com拿到AccessKey
5. **用户电脑WSL2里的智能体=Linux环境** — WSL2和Windows桌面有鸿沟，不能直接操控桌面GUI软件
6. **Claude Code/Llama CLI等智能体工具都是命令行工具** — 它们本身不能操作GUI，需要pyautogui中间层
7. **要分清"服务器能做"和"本地Windows能做"** — 如果智能体在服务器（Linux），那桌面GUI更没戏

## 案例：剪映视频效果提升

**实际案例参考：**

用户想让智能体操作剪映提升视频画质/字幕/调色效果。

### 错误做法
```
"你开剪映，我一步一步教你调"
→ 用户反感：我找你就是不想自己动手
```

### 正确做法（产出的文件）
```
我在服务器上：
✅ 生成LUT调色预设（.cube文件）→ 你导入剪映一键套用
✅ 用Python做封面图 → 你下载导入当封面
✅ 生成字幕样式参数 → 你在剪映设一次，以后套用
✅ 用即梦API生成画面片段 → 你下载导入时间轴
```

### 进阶做法（全自动出片）
```
用户：录口播（手机/电脑1分钟）→ 传音频文件到服务器
服务器：FFmpeg调色 → 加LUT → 做字幕 → 即梦生成片段 → 合成BGM → 输出成片
用户：下载成片 → 上传视频号（1分钟）
```
