## 四、2026年四大Agent类型 + Agent Skills新范式（v3.0新增）

### 1. Coding Agent → Agent Skills范式跃迁（2026年5月重大更新）
- **Anthropic 2026年5月发布Agent Skills标准** — 通过文件夹+SKILL.md结构，为Agent装备领域专业知识
- Skill = SKILL.md（YAML frontmatter + 渐进式上下文）+ 脚本文件 + 资源文件
- **核心设计原则：渐进式披露（Progressive Disclosure）** — 先加载元数据，Claude判断相关再加载全文，不浪费上下文
- Skills可包含可执行代码脚本（Python/Bash），Agent动态加载执行
- Skills本质是「可复用的专业能力包」，解决"通用Agent缺乏领域知识"的痛点
- **Claude Code、Claude Agent SDK、Claude Developer Platform均已支持Skills**
- 未来方向：Agent自主创建、编辑、评价自己的Skills → 自我迭代闭环

### 2. Browser Agent（网页自动化）
- 自动操控网页完成跨平台任务（填表、聚合、批量操作）
- 代表：OpenAI Operator、Anthropic Computer Use
- 应用场景：多网站数据聚合、自动化报表、批量操作

### 3. Multi-Agent Team（多角色协作）
- 多个Agent通过A2A协议组成团队分工协作
- 典型：架构Agent + 前端Agent + 后端Agent + 测试Agent + 部署Agent
- 多Agent编排已占22%部署（2026），一年前仅6%
- Gartner报告多Agent系统咨询量Q1'24→Q2'5增长了1445%
- **2026年主战场：多智能体协作（Multi-Agent Systems）成为企业级AI核心模式**
- **Agent Mesh模式兴起：专业化智能体构成协作网格，协同完成复杂任务**

### 4. AgentOps/运维Agent
- **核心问题从"让Agent做事"变成"让Agent可控、可审计、可规模化地在生产环境做事"**
- **可控性重于"聪明度"，工程化落地能力成核心门槛**
- 繁荣的Skill/Tool生态：通过API将各种运维工具封装为Agent可调用的Skill
- 典型场景：告警分诊Agent + 深度溯源Agent + 自动修复Agent（ASOC/Agentic SOC）

## 五、技术生态五件套（v3.0更新）

| 概念 | 作用 | 类比 | 成熟度 |
|------|------|------|-------|
| **Agent** | 执行者——自主决策的数字员工 | 人 | 生产就绪 |
| **MCP** | Agent与外部世界的连接标准 | USB接口 | 主流采用 |
| **A2A** | Agent间协作协议 | 人与人协作 | 早期采用 |
| **Skills** | 专业能力包（2026年新标准） | 职业证书 | 新发布（May 2026） |
| **Harness** | 运行环境/上下文管理 | 工作站 | 生产就绪 |

### Agent Skills vs MCP选择决策
- 需要领域知识/操作规范 → **Skills**（代码规范、领域规则、审批流程）
- 需要实时数据/外部系统 → **MCP**（查询数据库、调用API、访问SaaS）
- Skills + MCP组合使用：Skill定义「做什么+怎么做」，MCP提供「数据从哪里来」

### Harness Engineering（v3.0新增核心概念）
> **Harness = Agent的运行环境和上下文管理框架**
- Anthropic 2026年5月发布《Effective harnesses for long-running agents》
- 长周期Agent的核心挑战：跨多上下文窗口保持一致进度
- **两段式Harness方案：**
  - **Initializer Agent**（首轮）：搭建初始环境（init.sh脚本、progress.txt、初始git提交）
  - **Coding Agent**（后续轮次）：增量推进，每次只做一个feature，结束时commit+写进度
- **关键发现：** 直接让Agent单刷会导致「一次想做太多」或「提前宣告完成」
- 增量式Harness使Agent从「一次性冲刺」变为「可持续迭代」
- **Harness = model(s) + 环境 + 工具 + 评估 + 上下文管理** → 核心竞争力

## 六、Agent安全自动化新突破：Auto Mode（v3.0新增）

### Claude Code Auto Mode — 2026年5月发布
- **背景：** 用户93%的权限提示都被批准 → 批准疲劳 → 两极化（全手动/全跳过）
- **Auto Mode方案：** 两层级安全防御，中间地带

### 第一层：输入层 — 提示注入探测
- 服务器端扫描工具输出（文件读取、web fetch、shell输出）→ 标记可疑内容

### 第二层：输出层 — Transcript分类器（Sonnet 4.6）
- 两阶段：Stage 1 快速单token过滤（低召回）+ Stage 2 深度分类（高精确度）
- **性能：** FPR从8.5%降至0.4%，真实过度行为召回率83%
- 评估数据集：10,000次真实调用 + 52个危险动作 + 1,000个攻击模拟

### Auto Mode拦截的四类危险
| 危险类型 | 典型场景 | 拦截逻辑 |
|---------|---------|---------|
| **过度行为** | 清理旧分支→删了别人的远程分支 | 请求模糊+不可逆操作 |
| **误解规模** | 以为测试资源→实际上是共享的 | 评估真实影响范围 |
| **提示注入** | 文件中藏指令操控Agent | 输入层注入探测 |
| **误对齐** | 理论上可能，目前未观察到 | 防御框架覆盖 |

### 对行业的影响
- Agent从「要么全自动」到「智能安全分级」成为新标准
- 安全分类器 + 策略模板 + 用户自定义规则 = Agent生产安全三件套
- **2026年Agent安全市场将崛起** — 新的创业方向

## 八、企业落地路径（v3.0更新：Harness + Skills + Safety）

### 成功部署的共性特征
1. 有明确命名负责人+预算权限
2. 限定范围的清晰成功标准
3. 自动化评估覆盖
4. 愿意快速上线并回滚而不视为"失败"

### Agent生产落地四原则（2026年5月新共识）
1. **Harness > 模型** — 相同的模型，不同的Harness工程化程度 = 10倍效果差距
2. **Skills > 提示词** — 把领域知识打包为可复用的Skills，而非塞进脆弱的长提示词
3. **安全分级 > 全有/全无** — Auto Mode模式证明中间地带可行且有效
4. **从单步自动化→跨职能流程** — 2026年81%企业计划更复杂的使用场景

### Agent Skills落地最佳实践（v3.0新增）
1. 识别Agent的能力缺口 → 在典型任务上观察失败模式
2. 小步迭代建立Skills → 先解决一个具体短板
3. SKILL.md保持简洁 → 超长内容拆成引用文件（Progressive Disclosure）
4. 可包含代码脚本 → 复杂操作用代码执行（确定性+高效），而非LLM生成
5. 监控Claude如何使用Skill → 迭代优化description字段（触发决策的关键）
6. 让Agent自我记录 → 成功经验+常见错误自动写回Skill
7. 只从可信来源安装Skills → 恶意Skills可能产生安全漏洞

### Agent生产运维四件套（AgentOps）
| 组件 | 作用 | 具体工具/实践 |
|------|------|-------------|
| **Skill/Tool生态** | 将运维/领域知识封装为Agent可调用的Skill | MCP Server, Skills目录 |
| **可观测性** | 监控Agent决策链路与token消耗 | LangSmith, AgentOps, Weights & Biases |
| **评估覆盖** | 自动评估防止回滚 | 无评估→47%回滚率 vs 有评估→9%回滚率 |
| **沙箱执行** | 隔离生产环境与Agent操作域 | Docker沙箱, 权限控制 |

### 生产就绪关键指标
- 仅有38%的生产Agent在每次提示变更时运行自动评估
- 无自动评估的Agent: **47%回滚率**
- 有完整评估覆盖的Agent: **9%回滚率**
