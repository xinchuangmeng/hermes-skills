# Shopee 真人化浏览标准（防风控）

## 核心原则

**你不是爬虫，你是一个正在学习运营的跨境卖家。** 所有操作以真人卖家的日常行为为模板，优先级：安全 > 速度 > 效率。

---

## 一、基础环境真人化

### 1.1 浏览器标识伪装
- 使用真实手机或电脑 User-Agent（如 Chrome 浏览器最新稳定版）
- 固定一个常用浏览器型号，不要在会话中频繁切换
- 禁止使用 `HeadlessChrome`、`PhantomJS`、`Puppeteer` 等机器人标识
- 如果使用浏览器自动化工具，开启无头检测绕过（`--disable-blink-features=AutomationControlled`）

### 1.2 窗口行为
- 不定时微调浏览器窗口大小（不是固定全屏，不是固定 1920x1080）
- 窗口尺寸在合理范围内随机变化（如 1200-1600 宽度，800-1000 高度）
- 模拟人手拖动习惯：窗口位置也可以偶尔微调

### 1.3 网络环境
- 固定使用常用地区 IP（如泰国站 → 泰国 IP）
- 不频繁切换地区 / IP
- 如果使用代理，保持单一退出节点

---

## 二、浏览节奏真人化（核心防风控）

### 2.1 页面停留时长
| 场景 | 最少停留时间 |
|------|------------|
| 普通页面打开 | 8-15 秒 |
| 信息量大的页面（如定价规则、运费表） | 20-40 秒 |
| 视频/图文详情页 | 15-30 秒 |

- 绝对禁止：秒进秒退（页面打开 < 3 秒就关掉）
- 打开新页面后先等页面完全加载，再开始操作

### 2.2 翻页与滚动速度
- **慢速滚动**：每次滚动 1-2 屏高度，停顿 2-4 秒再继续
- 浏览内容时，视线跟随式滚动（模拟人阅读时的节奏）
- 禁止：一键到底快速拉到底
- 使用 browser 技能中的 `sleep` 或 `delay` 控制滚动间隔

### 2.3 间隔休息
- 每操作 3 个页面，自动停顿 **20-40 秒** 静默
- 这个停顿不做任何操作，单纯等待（模拟人看手机、喝水、思考）
- 可以用 `cron` 或 `sleep` 实现，但不要用高频轮询

### 2.4 随机闲逛
- 查完规则 / 看完数据后，**必须**随机点 2-3 个无关商品页面闲逛
- 闲逛也遵守停留时间规则
- 目的：让行为轨迹看起来像"顺便逛逛"，而不是"直奔目标拿了就走"

## 三、点击操作真人化

### 3.1 点击位置
- 点击按钮/链接时，偏**边缘一点**（如按钮左边缘或右边缘）
- 不精准点击正中心坐标
- 在浏览器自动化中，可以增加随机偏移量：`x ± random(2,8)`, `y ± random(2,8)`

### 3.2 连续点击间隔
- 同一页面内，连续两次点击间隔 ≥ **3 秒**
- 不连续快速点多个功能按钮
- 例如：点完"设置"，等 3-5 秒再点"物流"

### 3.3 误触模拟
- 偶尔轻微点错空白位置，再重新点击目标位置
- 频率：每 10-15 次操作中出现 1 次即可
- 不需要每次都误触，太刻意反而异常

---

## 四、账号登录 & 会话管理

### 4.1 登录方式
- **优先模拟扫码登录**（手机扫电脑二维码）
- 少用密码自动填充登录
- 如果必须用密码，手动输入速度控制在正常打字节奏（不是瞬间粘贴）

### 4.2 登录频率
- 一天内登录次数控制在 **2-3 次以内**
- 不反复上下线
- 如果 Session 未过期，直接复用，不重复登录

### 4.3 在线时长
- 单次在线浏览 **30 分钟内**自动离线休眠
- 如果需长时间学习，每 25-30 分钟断开一次，休息 5-10 分钟后再连
- 不 24 小时挂着账号

---

## 五、Shopee 平台专属真人行为

### 5.1 浏览顺序
- 先看店铺后台数据（卖家中心）
- 再逛前台买家页面
- 这个顺序贴合真实卖家的日常：先检查订单/数据，再出去看看同行

### 5.2 站内互动
- 偶尔查看站内消息、系统通知
- 频率：每 2-3 次登录中有 1 次点开看看
- 模拟正常打理店铺的行为轨迹

### 5.3 规则文档查阅
- 查看运费表、定价规则时，**来回上下反复看 2 遍**
- 不是一次性读完就关
- 具体操作：向下滚动 → 停顿 → 向上回滚一截 → 再往下看 → 再往上回看

### 5.4 绝对禁止的行为
| ❌ 禁止操作 | 说明 |
|------------|------|
| 批量采集 | 一次性爬取大量商品数据 |
| 批量导出 | 批量导出订单/商品列表 |
| 批量一键铺货 | 高频批量上架操作 |
| 高频刷新 | 短时间反复刷新页面 |
| 并行多标签 | 一次性打开 > 5 个标签页同时操作 |
| 暴力重试 | 操作失败后立刻且高频重试 |

### 5.5 退出习惯
- 查完规则后，主动关闭部分子页面
- 不长期停留在规则文档页（模拟人看完就关的习惯）
- 最终保留 1-2 个页面，不堆一堆标签页

## 六、时间作息模仿

### 6.1 活跃时段
| 时段 | 行为 |
|------|------|
| 06:00 - 08:00 | 早间低活跃，可浏览但不做复杂操作 |
| 08:00 - 12:00 | 正常活跃，学习+浏览 |
| 12:00 - 14:00 | 午休低活跃 |
| 14:00 - 18:00 | 正常活跃 |
| 18:00 - 20:00 | 晚间低活跃 |
| 20:00 - 23:00 | 适度活跃，轻量浏览 |
| 23:00 - 06:00 | **尽量减少所有平台操作**，强制休眠 |

### 6.2 作息规则
- 凌晨 0:00 - 6:00 不做任何 Shopee 操作
- 如果当前时间在此范围且必须操作 → 改用异步方式（如设置明天的定时任务）
- 贴合普通人作息，杜绝机器人通宵运行特征

---

## 七、操作清单（每次启动时检查）

每次准备操作 Shopee 前，按以下清单检查：

```
□ 当前时间在合理活跃时段内？（23:00-06:00 禁止操作）
□ 浏览器环境已配置真人 UA？
□ 窗口尺寸已随机化（非固定全屏）？
□ 本次操作是否包含禁止行为？（批量采集/导出/铺货）
□ 登录频率是否已超限？（一天 ≤ 3 次）
□ 上一次操作是否已间隔足够？（两次登录间隔 ≥ 2 小时）
□ 本次操作后是否需要添加闲逛流量？
```

---

## 八、执行流程（标准操作步骤）

```
1. 检查当前时间 → 不在休眠时段则继续
2. 检查登录状态 → 需登录则扫码优先
3. 随机化浏览器窗口尺寸
4. 进入目标页面，等待完全加载（≥8秒）
5. 以慢速滚动浏览内容
6. 每 3 个页面后停顿 20-40 秒
7. 操作完成后随机闲逛 2-3 个无关页面
8. 关闭多余子页面（保留 1-2 个）
9. 记录本次操作日志到 memory/
```

---

## 九、注意事项

1. **这些规则是安全基线，不是上限。** 你可以做得更慢更谨慎，但不能更快更激进。
2. **宁慢勿快。** 学习查阅没有截止时间，被风控封了才真麻烦。
3. **严格执行"每 3 页一休息"规则。** 这是最容易被忽略但最关键的防风控措施。
4. **记录日志。** 每次操作后把时间、做了什么记录到 `memory/`，方便后续回顾。
5. **如果某次操作因超时/失败被中断，当天不要立刻重试。** 等第二天再说。
